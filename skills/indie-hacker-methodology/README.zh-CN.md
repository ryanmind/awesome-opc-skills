## 🎯 概述

本方法论总结了成功独立开发者的核心原则：

- 快速发布（2周内完成 MVP）
- 从第一天就收费（最低 $30/用户）
- 保持独立（不拿风投）
- 公开构建（分享你的旅程）
- 使用已有技术（不要先学新技术）

## 🚀 何时使用

当你需要以下指导时激活此技能：

- **产品验证** - "如何验证我的想法？"
- **定价策略** - "应该收费多少？"
- **技术选型** - "应该学 React 还是用我会的工具？"
- **融资决策** - "要不要融资？"
- **增长策略** - "如何获取第一批用户？"
- **独立开发者心态** - "一个人真的能做到吗？"

## 💡 核心原则

### 1. 快速发布
- 95% 的项目会失败，所以要快速构建和迭代
- 2周完成 MVP，而不是6个月
- 完成比完美更重要

### 2. 从第一天就收费
- 最低 $30/用户
- 不做免费增值，不做免费试用
- 付费验证真实需求

### 3. 保持独立
- 不拿风投
- 保持 90% 利润率
- 保持你的自由

### 4. 公开构建
- 在社交媒体上分享你的旅程
- 记录成功和失败
- 社区驱动增长

### 5. 使用已有技术
- Google 表格就足够开始
- 不要为了开始而学新技术
- 用现有技能发布

## 📚 示例

### 验证产品想法

**问题**: "我想做一个远程工作招聘平台，需要先做市场调研吗？"

**答案**: 不需要市场调研。2周内发布：
1. 用 Google 表格列出 10-20 个职位
2. 添加 Stripe 支付（每个职位 $50-100）
3. 用 Carrd 做一个简单的着陆页
4. 如果有公司付费 → 需求存在，继续做
5. 如果没人付费 → 调整方向或定价

### 定价策略

**问题**: "要不要提供免费版本？"

**答案**: 不要。从第一天就收费：
- 最低 $30/用户（越高越好）
- 免费用户制造噪音，不产生收入
- 付费是最强的需求信号
- 目标 90% 利润率以保持可持续性

### 技术选型

**问题**: "我应该为项目学 Next.js 吗？"

**答案**: 使用你已经会的工具：
- 如果你会 React → 用 React
- 如果你不会编程 → 用 Google 表格 + Stripe
- 先发布，后优化
- 技术债不是问题，没有用户才是问题

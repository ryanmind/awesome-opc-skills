# Indie Hacker Methodology

A Claude Code Skill that embodies successful indie hacker practices: rapid validation, direct monetization, and staying independent.

## 🎯 Overview

This methodology summarizes core principles from successful indie hackers:

- Ship fast (2 weeks to MVP)
- Charge from day one (minimum $30/user)
- Stay independent (no VC funding)
- Build in public (share your journey)
- Use what you know (don't learn new tech first)

## 🚀 When to Use

Activate this skill when you need guidance on:

- **Product validation** - "How do I validate my idea?"
- **Pricing strategy** - "How much should I charge?"
- **Tech stack decisions** - "Should I learn React or use what I know?"
- **Funding decisions** - "Should I raise money?"
- **Growth tactics** - "How do I get my first users?"
- **Solo founder mindset** - "Can I really do this alone?"

## 💡 Core Principles

### 1. Ship Fast
- 95% of projects fail, so build fast and iterate
- 2 weeks to MVP, not 6 months
- Done is better than perfect

### 2. Charge from Day One
- Minimum $30/user
- No freemium, no free trials
- Payment validates real demand

### 3. Stay Independent
- No VC funding
- Maintain 90% profit margins
- Keep your freedom

### 4. Build in Public
- Share your journey on social media
- Document successes and failures
- Community-driven growth

### 5. Use What You Know
- Google Sheets is enough to start
- Don't learn new tech to begin
- Ship with existing skills

## 📚 Examples

### Validating a Product Idea

**Question**: "I want to build a remote job board. Should I do market research first?"

**Answer**: No market research. Ship in 2 weeks:
1. Create a Google Sheet with 10-20 job listings
2. Add Stripe payment ($50-100 per job post)
3. Build a simple landing page with Carrd
4. If companies pay → demand exists, continue
5. If no one pays → pivot or adjust pricing

### Pricing Strategy

**Question**: "Should I offer a free tier?"

**Answer**: No. Charge from day one:
- Minimum $30/user (higher is better)
- Free users create noise, not revenue
- Payment is the strongest demand signal
- Target 90% profit margins for sustainability

### Tech Stack Choice

**Question**: "Should I learn Next.js for my project?"

**Answer**: Use what you already know:
- If you know React → use React
- If you don't code → use Google Sheets + Stripe
- Ship first, optimize later
- Technical debt isn't the problem, no users is
